## What Was Fixed

### The Problem
```
ERROR: "failed to lazily initialize a collection of role: Quiz.questions: 
       could not initialize proxy - no Session"
```

When you called `GET /quizzes`, the application threw a LazyInitializationException because Hibernate tried to lazy-load the questions after the database session had closed.

### The Solution (Implemented)
✅ **@Transactional** - Keeps Hibernate session open during JSON serialization  
✅ **LEFT JOIN FETCH** - Loads questions in a single database query  
✅ **@JsonManagedReference/@JsonBackReference** - Prevents circular reference in JSON

## Key Changes Summary

### 1. Keep Hibernate Session Open
```java
@Transactional(readOnly = true)  // ← ADD THIS
public List<Quiz> getAllQuizzes() {
    return quizRepository.findAllWithQuestions();
}
```

### 2. Load Everything in One Query
```java
@Query("SELECT DISTINCT q FROM Quiz q LEFT JOIN FETCH q.questions")
List<Quiz> findAllWithQuestions();
```

### 3. Prevent Circular References in JSON
```java
// Quiz.java
@JsonManagedReference("quiz-questions")
List<Question> questions;

// Question.java
@JsonBackReference("quiz-questions")
Quiz quiz;
```
